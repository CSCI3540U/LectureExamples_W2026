Web Security Demonstrations

Select a vulnerability below to explore an intentionally insecure demo.

XSS

Cross-Site Scripting

Attacker-controlled input is rendered as executable HTML/JavaScript inside another user's browser.

  • Reflected XSS — malicious payload lives in the URL; the server echoes it back immediately.
  • Stored XSS — payload is saved in the database and served to every visitor who loads the page.
Try pasting <script>alert('XSS')</script> or <img src=x onerror="alert('XSS')"> into the inputs.
CSRF

Cross-Site Request Forgery

A malicious page tricks an authenticated user's browser into making an unintended state-changing request to another site.

  • Victim bank page — a simple transfer form with no CSRF protection.
  • Attacker's evil page — auto-submits a hidden form to the bank the moment it loads.
Open the bank page first, note your balance, then visit the evil page in the same browser — watch the balance change automatically.
SSRF

Server-Side Request Forgery

The server is tricked into making HTTP requests on the attacker's behalf, potentially reaching internal services.

  • SSRF demo — the server fetches any URL you supply.
  • Internal secrets endpoint: /internal/secrets (normally unreachable from outside).
Try fetching http://127.0.0.1:5000/internal/secrets through the SSRF form to access data that should be private.
These pages are intentionally vulnerable. Run this app in an isolated environment only. Never expose it to a network.

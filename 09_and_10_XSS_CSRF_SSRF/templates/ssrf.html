SSRF — Server-Side Request Forgery

How it works: The application accepts user input and uses it to construct a URL that the server then fetches on your behalf. If that URL isn't validated, an attacker can point it at internal services that are unreachable from the public internet.
Protection mode: Vulnerable Protected: no outbound fetch

fetched_url = request.form.get("url", "").strip()

{% else %} Protected code 
# app.py — user input never influences an outbound request
ticker = request.form.get("ticker", "").upper()
data = STOCK_PRICES.get(ticker)   # ← local dict lookup, no HTTP request
if data:
    result = json.dumps({"ticker": ticker, **data})
{% endif %}

The root cause of SSRF is allowing user-controlled input to determine what URL the server fetches. The fix is to never make that substitution at all.
